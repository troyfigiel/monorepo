#+title: Semantic versioning
#+filetags: :stub:

A semantic version number consists of a major version, minor version and patch number.

- Patches are used for bug fixes and do not change the functionality of the code
- Minor changes are used for functionality changes without breaking the current [[id:622bb467-39cf-4839-bcce-cbd7bfec5be5][API]]
- Major changes are changes that do break the current [[id:622bb467-39cf-4839-bcce-cbd7bfec5be5][API]].

The first version starts at 0.1.0 and the first [[id:4a2f8a39-9750-445b-b450-2b4964895913][Stable version]] is version 1.0.0.
